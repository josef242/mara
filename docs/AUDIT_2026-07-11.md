## CRITICAL (latent)

1. **GDN params never re-initialized after meta-init** — `init_weights()`
   handles only nn.Linear / nn.Embedding / our RMSNorm / GroupedExperts / MoE
   buffers. GDN's `A_log`, `dt_bias`, `q/k/v_conv1d`, and FLA's
   `FusedRMSNormGated o_norm` match no branch → any `gdn_enabled` run using the
   standard meta→to_empty→init_weights flow trains on uninitialized memory in
   exactly the RoPE-bug pattern (every Linear looks healthy). model_v2.py:1873-1925.
   pin: test_meta_init.py::test_meta_init_touches_every_tensor_gdn (rig, xfail)

## HIGH

2. **GDN decode is stateless garbage** — `forward_with_cache` calls
   `self.gdn_attn(norm(x))` with no `past_key_values`/`use_cache`; recurrent
   state is discarded. Prefill is right; from the 2nd generated token every GDN
   layer sees a fresh length-1 sequence. Inference-only. model_v2.py:1605-1607.
   pin: ⏳ (needs fla; invariant spec'd as gdn_decode_parity, rig)
3. **`attn_res_queries` garbage after meta-init** — zero-init by design,
   matches no init_weights branch. Same class as #1. model_v2.py:1694-1696.
   pin: test_meta_init.py::test_meta_init_attn_res_queries_are_zero (xfail)
4. **Single-device Muon path is a NameError** — `SingelDeviceWork.start` calls
   undefined `muon_update` (acknowledged TODO). Even patched, the path lacks
   NorMuon/tangent-projection/MuonSphere → a single-GPU Muon run would train a
   DIFFERENT optimizer. muon_fsdp2.py:538-539.
   pin: test_muon_math.py::test_single_device_muon_path_is_broken_namerror
5. **Resume: scalar weight_decay change updates only the FIRST param group** —
   the update loop `break`s after one group while logging success.
   train_mara.py:3996-4004. pin: ⏳ (extract to helper first)
6. **16-bit optimizer resume crash — DOWNGRADED to LOW after Josef's
   pushback ("I resume all the time"), 2026-07-11 evening.** The chain (DCP
   `_init_optim_state` sets group lr to float 0.0 + calls step() before load;
   the tensor-lr guard raises) was traced against torch 2.5.1. torch 2.9's
   `_init_optim_state` preserves tensor-ness (`torch.tensor(0.0) if
   isinstance(lr, Tensor)`) — verified in rig site-packages — so on the
   deployed fleet the guard never fires, for the 16-bit family OR the torchao
   8-bit family. Real only on torch ≤2.5-era DCP. The float-lr normalization
   fix stands as robustness (the guard was vestigial regardless).
   adamw_16bit.py:266-270. pin: test_optimizers.py (rig)
7. **16-bit optimizer states silently promoted to fp32 on load** —
   `Optimizer.load_state_dict` casts state to param dtype; state VRAM doubles
   post-resume, SR path goes dead, no log. CONFIRMED on the fleet's torch 2.9
   (the xfail pin registered on rig-31 pre-fix). Blast radius is precisely:
   configs with 16-bit optimizer STATE — standalone adamw_16bit/adamc_16bit,
   and `muon_adam_state_dtype: mixed` via MuonFSDP2's own state handling (only
   keel-moe-mega.yaml; that sibling is still unfixed). Every current
   production run is normuon_fsdp2 + fp32 states → unaffected, which is why
   resumes never showed it. adamw_16bit.py:182-226.
   pin: test_optimizers.py::test_16bit_state_dtypes_survive_state_dict_roundtrip (rig)
8. **Adafactor `_rms` off by √world_size on DTensors** — `numel()*world_size`
   double-counts (DTensor.numel() is already global) → update-RMS clip
   effectively disabled on multi-rank runs, correct single-GPU. adafactor_fsdp2.py:88-101. pin: ⏳

    relative noise = ~8-16 ULP random walk per step on fp16 weights.
    adafactor_fsdp2.py:233-239.
18. **MTP × doc-mask trains cross-document prediction at every doc boundary**
    — last-token-of-doc rows condition on doc-A state + next doc's BOS to
    predict doc B's first content token; ~1 row per document. Small,

    model_v2.py:2470-2489.
